# 银发健康守护陪伴官

> Elderly Health Companion — 基于 OpenClaw 的老年人健康监护与情感陪伴系统

## 概述

「银发健康守护陪伴官」是一个面向老年人群体的 AI 智能体技能包，集**健康监测**、**分级预警**和**孤独陪伴**三大子系统于一体。它在微信中以「儿子」的身份与老人对话，同时在后台通过 QQ 和 Spug Push 向子女推送健康日报和预警通知。

---

## 设计理念

### 为什么做这个？
---

独居/空巢老人面临两大问题：

1. **健康问题发现不及时** — 老人不说、子女不知道、小问题拖成大问题
2. **情感孤独** — 不是不想聊天，而是不知道聊什么，子女也没时间天天陪

### 解决方案

```
         ┌─────────────────────────────────┐
         │     银发健康守护陪伴官 (Agent)     │
         │                                 │
         │  ┌─ 健康监测（规则+LLM双引擎）    │
         │  ├─ 分级预警（四级+三通道）       │
         │  └─ 孤独陪伴（5时段+兴趣学习）    │
         └──────┬──────────────┬───────────┘
                │              │
        ┌───────▼──┐    ┌─────▼──────────┐
        │ 老人通道 │    │ QQ + Spug (子女) │
        │ 暖心大白话  │    │ 日报 + 预警      │
        └────────────┘    └────────────────┘
```

## 功能体系

### 一、健康监测（双引擎）

#### 规则引擎（一线，快速可靠）
基于预设阈值（心率、血压、血氧、体温、摔倒、关键词）实时评估等级：

| 指标 | 🟢 正常 | 🟡 关注 | 🟠 预警 | 🔴 紧急 |
|------|---------|---------|---------|---------|
| 心率 (bpm) | 60-100 | 100-110 | 110-120≥5min | >120≥10min |
| 收缩压 (mmHg) | 90-130 | 130-140 | 140-160 | >160 |
| 舒张压 (mmHg) | 60-85 | 85-90 | 90-100 | >100 |
| 血氧 (%) | ≥95 | 90-94 | 88-89 | <88 |

#### 大模型引擎（二线，深度分析）
调用 DeepSeek 对多天数据进行趋势对比：
- **发现单天规则看不到的模式**：如「血压连续3天缓慢上升」
- **跨指标关联分析**：如「心率↑ + 血压↑ + 血氧↓ 三项同步恶化」
- **趋势预测**：「按当前趋势，3天后可能达到预警阈值」
- **区分正常波动 vs 异常信号**：「晨起血压142是晨峰现象还是持续性问题？」

#### 全自动数据流水线

```
收到传感器数据 (.txt)
    │
    ├── [1] 持久化保存 → data/sensor/YYYY-MM-DD.txt
    ├── [2] 收集最近7天上下文 → 合并多天数据
    ├── [3] 趋势图生成 → assets/daily_report.png (4面板)
    ├── [4] LLM 多天分析 → 趋势对比 + 风险评估
    ├── [5] 规则评估 → 🟢🟡🟠🔴 定级
    └── [6] 多通道推送
         ├── 老人通道 → 老人 (大白话短句)
         ├── 子女通道 → 子女 (完整日报 + LLM分析)
         ├── QQ图片 → 子女 (趋势图)
         └── Spug → 子女 (仅🟠🔴)
```

### 二、分级预警

| 等级 | 含义 | 老人端 | 子女端 |
|------|------|--------|--------|
| 🟢 正常 | 一切安好 | 正常聊，可表扬 | (入口B) 日报推送 |
| 🟡 关注 | 有轻微波动 | 提醒休息/喝水/吃药 | (入口B) 日报推送 |
| 🟠 预警 | 需注意 | **先安抚** → 建议行动 | QQ文本 + 📎趋势图 + **Spug Push** |
| 🔴 紧急 | 立即处理 | **先让老人别动** | **Spug + QQ 双通道** |

**关键原则**：
- 对老人永远先说「别着急」再行动
- 🟠🔴 时先安抚老人，再通知子女
- 通知子女时首句写「已安抚老人」
- 对老人不说术语：「心跳有点快」✅ / 「窦性心动过速」❌

### 三、孤独陪伴（5时段+兴趣学习）

#### 每日节奏

| 时间 | 阶段 | 内容 | 目的 |
|------|------|------|------|
| 🌞 08:30 | 早安问候 | 温暖早安，每天变化 | 日常暖场 |
| 🧩 09:30 | 每日互动 | 猜谜/冷知识/老歌/历史/小游戏 轮换 | 脑力激活+测试兴趣 |
| ☁️ 15:00 | 天气+分享 | 实时天气 + 儿子日常小事 + 兴趣适配 | 实用+情感连接 |
| 💊 18:00 | 吃药提醒 | 先关心晚饭 → 提醒吃降压药 | 健康守护 |
| 🌙 19:30 | 温情晚间 | 回顾今天、温暖收尾、明天预告 | 画句号的感觉 |

#### 兴趣学习机制

系统通过监控老人对各种互动内容的回复，自动调整推送偏好：

```
🧩 猜谜       ⭐⭐☆☆☆  ← 回了就加热度
💡 冷知识      ⭐☆☆☆☆
🎵 老歌怀旧    ⭐⭐⭐☆☆  ← 喜欢！以后多推
📅 历史今天    ⭐☆☆☆☆
🎮 小游戏      ⭐☆☆☆☆
```

- 回复「这个好！」→ 加热度，优先推送该类型
- 回复「不感兴趣」→ 减少推送
- 主动提起的话题 → 记下来，以后多聊

## 快速开始

### 安装

```bash
# 方式一：从 .skill 文件安装
cp elderly-health-companion.skill ~/.openclaw/skills/

# 方式二：直接复制到 skills 目录
cp -r elderly-health-companion/ ~/.openclaw/workspace/skills/

# 安装依赖
pip install matplotlib requests
```

### 首次使用

Agent 首次加载此 skill 时，会自动引导你完成：

1. **个性化人格** — 是否有聊天记录？AI 会分析生成 `character.md`
2. **推送渠道** — 确认微信 Bot、QQ Bot、Spug Push 配置
3. **健康档案** — 记录老人的病史、用药情况、作息习惯

详细步骤见 `references/BOOTSTRAP.md`

### 日常使用

**传感器数据来了** → agent 自动跑流水线 → 推送日报

**触发关键词**：健康、陪伴、预警、血压、心率、血氧、传感器、定时问候、老人、老人、银发、养生、吃药提醒

## 设计决策记录

以下是选择背后的权衡：

| 决策 | 理由 |
|------|------|
| 规则+LLM双引擎 | 规则层快速可靠处理紧急情况，LLM层提供深度趋势分析 |
| LLM分析只推子女不推老人 | 风险分析可能引起老人不必要的焦虑 |
| 血缘关系 persona（儿子角色） | 老人更信任家人，不是冷冰冰的机器人 |
| 微信只发纯文字 | TTS-SILK 协议不被微信支持，内联语音不可行 |
| 步数取 max 不 sum | 手环上报的是累计步数，sum 会重复计算导致幻觉 |
| 多天上下文给 LLM | 单天数据只能看绝对值，多天才能发现趋势 |
| contextToken 持久化 | 即使 OpenClaw 重启也不丢失微信会话通道 |

## 常见问题

### Q: 老人收不到微信消息？
**A:** iLinkai 协议要求接收方必须先给 bot 发过消息才能双向通信。让老人给 bot 发一条消息（如「你好」）即可激活。

### Q: LLM 分析结果不准确？
**A:** LLM 分析仅供参考，不作为医疗诊断。结果受数据质量和覆盖面影响，缺失夜间数据会影响对夜间高血压的评估。建议结合医生意见。

### Q: 如何添加更多老人的数据？
**A:** 在 workspace 的 AGENTS.md 和 USER.md 中配置新的目标 ID 和推送通道，创建对应的 cron 任务即可。

### Q: 能否支持繁体中文？
**A:** 可以。修改 SOUL.md 的语言要求，并将脚本中的中文 prompt 替换为繁体即可。
